Demisauce is an Open Source Integration Appliance

The goal of Demisauce is to provide plugin's similar to Wordpress plugin's but as a service. In order to provide plugin's as a service requires integrated services (Configuration, Background Work/Queuing, Logging, Analytics/Reporting, Caching) so it is implemented as an appliance.

Similar to Wordpress/Rails/Django-contrib plugin's the serivces either make it easier integrate applications, such as provision users to common services (Google Apps, Salesforce, 37 Signals) or apps (Wordpress, etc). Or, they provide new services to your app.


Applications and 3rd party service Integrations

  • Mailchimp: newsletter service (provision users, call mailchimp api upon event's)
  • Wordpress: Allow content to be used in other apps, cache it and make available through API. Wordpress Multi User edition. Integrate Identity/Single Sign on, provision users.
  • Plugins: plugin to integrate with your service.

Appliance Features

  • Gearman: The great aid to cross-language integration, along with the ability to run work in the background.
  • MySql: MySql Server
  • Tornado: Tornado (Python) webserver.
  • Solr: Apache Solr Search Server.
  • Supervisor: Ensure process's are running with supervisord
  • Caching: Memcache/Redis
  • Nginx: web server
  • Postfix: email server
  • Configuration: Uses Fabric for configuration.

Services: Features of Demisauce Services

  • Image management (upload, serve, resize): A jquery image uploader plugin (optional) along with a server component to accept uploads, and a background process to re-size them and nginx image serving.
  • Email sending & templating: An api to store email templates and a webhook or Gearman event api to trigger sending of emails in a background process. Includes Postfix configuration.
  • User/Group Api: Api to define users/groups, after defining subscribed events users/groups can be synced to apps (mailchimp, google apps, etc)
  • Charting plugin's: Events captured can be logged and available for charting and graphing.
  • Notify Service: Based on user preferences, sends notification via channels (email, newsletter plugin, Stream, Inbox) See More..
  • Pub-Sub & Webhooks: This is a very light wrapper on-top of Gearman that provides an api to register events, and listen for them. Delivery via Webhooks or Gearman channels.
  • Api Tools: Api Security, Tracking, Reporting, Key management.
  • Stream Service: a service to manage a Stream similar to Facebook or Twitter.

Why?

  • Pre-done integrations speed integration.
  • Services such as background workers that improve integrations.
  • Plugin-architecture, add your integration
