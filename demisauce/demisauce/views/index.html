Demisauce is an Open Source Integration Appliance

It is made to support common integration needs of applications such adding a user to foreign system (Wordpress, Google Apps, etc) upon adding new user. It provides a Network oriented plugin architecture to accomplish this.


User Provising Plugins

Provides plugin's to provision Users to other systems upon your call to AddUser API. Altough LDAP or AD integration is a common way to integrate user with cloud SaaS apps (Google Apps, Salesforce, etc) you need to push, this aids the integration. More...

Network Plugin Architecture

Inspired by the Wordpress add_action(), do_action() Plugin architecture, a pub-sub type plugin system on top of Gearman. More ...

Polls

Although consumer oriented poll tools make it easy to embed polls with javascript into your site, this may not suit needs that do not want the polls public. Also, there is the page load time, and page flicker with javascript loading.

Activity Tracking

Easily embed single javascript file, and track your users behavior and provide embeddable visualizations for your applications. More ...

How it Works

A demisauce server (either in your data center, or hosted on Demisauce.com) would host a set of services, that you embed in your application with javascript, or using the PHP, or Python client API's. Additionally, the Demisauce server allows you to register additional service's, it is a plugin framework for services. There are examples in the demo of an externa PHP, or django app hosting services and being easily consumable from other applications.

Why?

  • The need for application capability across a suite of applications. ie: same help system on all apps at a company.
  • Many Apps fragment User data: A users comments across many apps are not visible. Building just one app isnt realistic.
  • Easy functionality: As consumer javascript embedded services, or wordpress style plugins have shown, there is a need for easy additional fucntionality.
  • Need for a service oriented plugin framework to provide coherency to ad-hoc integration.
